# Impeccable Design Context — Vending Machine Simulation

이 파일은 이 레포의 모든 디자인·polish 작업에 **기준점**이 됩니다. 새로운 화면을 만들거나 기존 화면을 다듬을 때 여기 적힌 원칙·톤·제약을 먼저 읽고 적용하세요.

---

## Design Context

### Users
**학습 교재용 시뮬레이션 관람자.** 이 제품은 Anthropic Vending-Bench의 현상을 교재·학습 자료로 전달하기 위해 존재합니다. 주 사용자는 "LLM 에이전트가 실제로 어떻게 행동하는가"를 이해하려 화면을 **읽는 학습자**이며, 턴을 넘기며 에이전트의 의사결정·실패·회복을 관찰·복기합니다.

핵심 행동은 **관람·독해**이지 조작·실시간 반응이 아닙니다. 즉:
- 한 턴을 끝내고 **일시정지 상태에서 화면을 해석**하는 시간이 대부분
- 스크린샷·수업 자료로 캡처될 가능성이 높음
- 학습 맥락이라 "밀도 낮고 예쁜" 것보다 "밀도 있지만 위계 있는" 쪽이 유용함

### Brand Personality
세 단어: **warm · mechanical · opinionated**

- **Warm**: 차가운 대시보드 미학이 아님. 따뜻한 크림/베이지 기반. 시스템이 친절하되 감상적이지 않음.
- **Mechanical**: 정밀한 정렬, 장식 없는 숫자, 모노스페이스의 **의도적** 활용, 기계적 정직함. "기술적으로 보이게"가 아니라 "실제로 기계다운" 정확함.
- **Opinionated**: 모든 정보를 동등하게 보여주지 않음. 이 턴에 **가장 드라마틱한 변화**가 무엇인지 디자인이 의견을 냄. 평등주의적 그리드 거부.

### Aesthetic Direction
현재 팔레트(크림 `#FAF7F2` · 테라코타 `#D4714E`)는 유지. 라이트 모드 전용. 다크모드 없음.

방향성:
- **교재·아카이브 인쇄물의 감성**: 오래된 기술 매뉴얼, 박물관 캡션 라벨, 과학 교재 도판. 화면에서 "읽을 거리"라는 인식을 줘야 함.
- **기계적 정직함 + 따뜻한 종이**: 숫자와 상태는 장식 없이, 배경은 따뜻한 종이처럼.
- **의견 있는 위계**: 타이포 크기·무게·여백·색의 대비를 **과감하게** 써서 읽기 순서를 강제.

Anti-references (명시적 거부):
- **AI 슬롭** 전반. 구체적으로:
  - 좌측 4px 색상 accent stripe
  - 그라디언트 텍스트
  - 큰 아이콘 + 헤더 + 설명문 카드가 반복되는 그리드
  - 모든 카드에 둥근 모서리 + 얕은 드롭섀도우
  - 모노스페이스를 "기술적 느낌" 조미료로 뿌리기
  - 네온 cyan/purple/블루 그라디언트 accent
  - shadcn 템플릿 복제본, Linear/Vercel 복제본 느낌

### Design Principles

**1. 위계가 디자인 전부다 (Hierarchy Über Alles)**
사용자의 1순위 불만은 "한 턴에 정보가 너무 많아 조잡하다"였다. 모든 polish는 이 문제를 먼저 통과해야 한다.
- 같은 시각적 무게로 모든 것을 표시하지 말 것
- 매 턴마다 "이번 턴의 주인공 정보"가 무엇인지 디자인이 결정
- 타이포 스케일 ≥1.25× / 여백 리듬 대비 / 색 대비 — 이 세 가지로 위계를 강제

**2. 점진 공개 (Progressive Disclosure)**
정보를 동시에 다 쏟지 않는다. 기본 상태는 가장 중요한 것만, 나머지는 확장·호버·드로어·탭으로 접어둔다.
- AgentLog Thinking은 긴 텍스트 → 기본 축약, 펼치기
- Email 본문은 리스트에 미리보기만
- 차트의 보조 데이터는 호버 시 공개
- "오늘 이 턴에서 실제로 변한 것"만 우선 시야에

**3. 기계적 정직함 (Mechanical Honesty)**
숫자·상태·로그는 있는 그대로. 장식도 둔화도 없음. 단 "차갑지 않게" — warm 바탕과 결합해 온기를 잃지 않는다.
- 숫자는 tabular-nums, 정렬된 상태로
- 상태 배지는 의미 정확, 치장 없이
- 에러·경고는 숨기지 말고 노란 배너 그대로 존중

**4. 의견 있는 기본값 (Opinionated Defaults)**
교재라면 "이걸 먼저 봐라"가 있어야 한다. 평등주의적 그리드 거부.
- 턴 결과 중 **변화 크기**가 큰 항목을 시각적으로 부각
- 읽는 순서가 설계된 레이아웃 (asymmetric OK)
- 중앙 정렬 만능주의 금지, 좌측 정렬 + 의도적 비대칭이 기본

**5. AI 슬롭 리트머스 테스트**
모든 컴포넌트 완성 후 묻는다: "이거 AI가 만든 거죠?"라고 누가 말했을 때 즉시 수긍할 것 같으면 실패. 다시 한다.

## Implementation Constraints

- **Stack**: Next.js 16 (App Router) / React 19 / Tailwind CSS 4 / 기본 Recharts
- **Language**: 한국어 UI (영어 단어는 짧은 레이블에 한정)
- **Theme**: Light only
- **Accessibility**: 강제 요구 없음 (WCAG 미준수 허용). 단 Design Principle 1(위계)의 결과로 대비는 자연스럽게 확보될 것.
- **기능 불변**: polish는 **기능을 깨지 않는 범위**에서만. 데이터 플로우·상태 흐름·API 호출 경로 유지.

## Current Pain Points (polish 대상 우선순위)

사용자 피드백 및 탐색 결과:
1. **화면당 정보 밀도 과다** — 3-column 전체가 한 번에 보이고 턴마다 모두 업데이트되어 시야에 혼잡
2. **시각 위계 빈약** — 카드·텍스트 크기·색 무게가 대체로 균일해서 "뭘 먼저 볼지" 신호가 약함
3. **타이포 무개성** — SF Pro 시스템 폰트만 사용 → mechanical 정체성이 약함
4. **Recharts 기본 스타일** — 재무 차트가 템플릿스러움
5. **자판기 시각화 비중 애매** — 12슬롯이 우측 고정인데 턴마다 변화가 작으면 공간 낭비, 클 때는 존재감 부족
